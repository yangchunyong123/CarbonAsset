# 碳资产管理系统 V2 需求文档

## 1. 文档说明
- 文档目的：基于当前项目原型文件（Axure 导出静态页面）梳理系统建设需求，作为后续产品设计、研发实现与测试验收的依据。
- 分析范围：`登录页面`、`碳画像总览`、`能碳总览`、`数据填报`、`核算配置`、`分析报告`、`排放底单`、`燃料品种管理`、`生产用料管理`、`能耗分析`、`用电分析`、`资产总览`、`配额管理`、`交易情况` 等页面。
- 说明：本文档根据原型可见信息提炼，部分规则与字段约束在实现前需与业务方二次确认。

## 2. 项目背景与目标
- 背景：企业需要对能耗与碳排放进行统一管理，满足碳核算、履约、预警、交易与经营分析需求。
- 总体目标：
  - 建立“数据填报-核算计算-报告输出-资产运营”的闭环能力。
  - 实现能源与碳数据一体化管理（煤、油、气、电、热等）。
  - 支撑碳配额管理、交易记录管理与碳资产可视化运营。
  - 提供多维分析（趋势、结构、排名、预警）和报表导出能力。

## 3. 用户角色与权限（建议）
- 系统管理员：组织/用户管理、字典与参数维护、权限配置、系统配置。
- 碳核算管理员：核算配置、排放因子管理、核算计算、报告生成。
- 能源管理员：能源品种管理、生产用料管理、能耗/用电分析。
- 碳资产运营人员：配额管理、交易管理、资产总览、预警处置。
- 管理层查看用户：查看总览、分析报告、关键指标、排名与趋势。

## 4. 业务范围与模块划分

### 4.1 认证与入口
- 登录页面：
  - 账号密码登录。
  - 自动登录、忘记密码入口。
  - 登录成功后进入系统主页/框架页。

### 4.2 碳画像模块
- 碳画像总览：
  - 企业碳效评分、碳排放额度、行业排名展示。
  - 快捷入口：碳核算填报、碳排放报告、碳排放查询、碳排放分析、碳排放预警、碳额度配置、碳模型配置、碳核算配置。
  - 日历、提醒事项（如排放因子讨论任务）。
  - 关键指标卡：能源消费总量、碳排放总量、强度指标及同比。
  - 碳排放趋势图、能源消耗结构图。

### 4.3 碳核算模块
- 能碳总览：
  - 指标总览：碳排放配额量、累计碳排放总量、能耗总量、绿色能源等。
  - 年份切换（今年/历史年份）与年度趋势展示。
  - 能耗结构占比（煤、气、油、电、热）及环图展示。
  - 强度分析与碳效排名（可按区域/行业/企业切换）。
- 数据填报：
  - 按能源类型与月份录入数据（煤/油/气/电/热及细分项）。
  - 支持年度切换、月度统计、计算触发、报告生成。
  - 显示计算结果提示（如“计算成功”）。
- 核算配置：
  - 行业模板配置（如化工等）。
  - 核算边界配置：化石燃料燃烧、购入电力/热力、过程排放、回收利用等。
  - 因子项配置：排放因子、低位发热值、碳氧化率、工艺参数等。
  - 支持保存、重置、二次确认。
- 排放底单：
  - 作为报告详情/底稿页，承载核算明细、基础信息、设备信息、监测方法等内容（原型文案较少，建议后续补齐字段与样式）。
- 分析报告：
  - 报告类型：排放报告、能耗报告、告警报告。
  - 报告筛选：类型、名称、年份等。
  - 列表管理：查看、编辑、删除、下载、批量删除、分页。

### 4.4 能源管理与统计分析模块
- 燃料品种管理：
  - 维护燃料品种主数据（名称、类型、形态、别名、因子参数等）。
- 生产用料管理：
  - 维护生产过程用料与核算关联信息。
- 能耗分析：
  - 用能趋势、累计能耗与定额比、超定额分析。
  - 年份切换、月度数据表、合计与百分比展示。
- 用电分析：
  - 用电成本分析（尖峰平谷维度）。
  - 用电详情分析（年度趋势、定额对比、超额分析）。

### 4.5 碳资产模块
- 资产总览：
  - 碳信用评分、企业碳配额进度、已使用/预计使用/剩余量。
  - 碳价行情（K 线：分时、五日、日K、周K、月K、1年）。
  - 平均/最高/最低单价与成交额数据展示。
  - 配额预警记录展示。
- 配额管理：
  - 展示总预算、总能耗、累计排放及同比。
  - 分能源配额卡片（油/气/煤/电/热）：配额、累计能耗、费用。
  - 年度趋势图（用热、用电等）与年份切换。
- 交易情况：
  - 交易总览：碳资产总额、平均卖出单价、平均买入单价、当日碳价区间。
  - 筛选条件：交易类型、挂牌时间、挂牌金额等。
  - 交易记录管理：查询、重置、新增、导出、批量删除。
  - 交易明细：交易量、单价、总价、交易平台、交易时间、状态（挂牌/买入/卖出）。
  - 分页与每页条数配置。

## 5. 核心业务流程

### 5.1 碳核算闭环流程
1. 维护核算配置（行业模板、因子参数、核算边界）。
2. 维护能源/用料主数据（燃料、生产用料等）。
3. 按月填报能源与生产数据。
4. 执行核算计算，产出排放结果与同比趋势。
5. 生成报告（排放/能耗/告警）并支持下载。
6. 进入资产模块进行配额评估和交易决策。

### 5.2 碳资产运营流程
1. 查看资产总览与配额进度、预警信息。
2. 按能源类别评估配额消耗与成本。
3. 结合市场碳价制定买入/卖出/挂牌策略。
4. 登记交易记录并跟踪成交情况。
5. 回溯年度统计与经营分析，优化预算与履约计划。

## 6. 数据需求（核心实体）
- 用户与组织：用户信息、角色、权限、组织层级。
- 能源与用料主数据：品种名称、分类、单位、热值、含碳量、氧化率、排放因子、生效时间。
- 核算配置：行业、核算边界、公式版本、参数映射、启用状态。
- 填报数据：年份、月份、企业/区域维度、能源消费量、成本、备注、填报状态。
- 核算结果：排放量、能耗量、强度指标、同比环比、计算批次号。
- 报告数据：报告类型、名称、周期、关键指标、附件、生成时间、版本。
- 配额数据：年度预算、已使用、预计使用、剩余额度、预警阈值。
- 交易数据：交易类型、状态、交易量、单价、总价、时间、交易所、经办人。
- 预警数据：预警类型、触发条件、等级、处理状态、处理记录。

## 7. 功能性需求明细（摘要）
- 通用列表能力：筛选、重置、分页、排序、批量操作、导出。
- 通用图表能力：趋势图、结构图、对比图、K线图，支持年份维度切换。
- 报告能力：模板化生成、在线预览、下载导出、历史版本保留。
- 数据校验能力：填报必填校验、数值范围校验、异常值提示。
- 审计追踪能力：关键操作日志（保存配置、计算、删除、交易新增等）。

## 8. 非功能需求（建议基线）
- 安全性：
  - 账号安全（强口令、登录失败策略、会话超时）。
  - 权限控制（基于角色的页面与数据权限）。
  - 敏感操作审计与可追溯。
- 性能：
  - 常规列表查询响应建议 ≤ 3 秒。
  - 常规图表渲染建议 ≤ 3 秒。
  - 月度计算任务建议支持异步处理与进度反馈。
- 可用性：
  - 支持主流桌面浏览器。
  - 核心页面布局一致、面包屑与导航统一。
- 可维护性：
  - 配置参数可版本化管理。
  - 指标与报表口径可配置化扩展。

## 9. 验收要点（首版）
- 登录及基础导航可用，模块路由完整。
- 数据填报可按年/月录入并保存。
- 核算配置可维护并可触发计算。
- 能碳总览、能耗分析、用电分析图表可展示真实计算结果。
- 报告可生成、查询、下载。
- 配额管理与交易记录具备完整查询与维护能力。
- 关键操作具备权限校验与日志留痕。

## 10. 风险与待确认事项
- 原型中部分页面存在演示数据与重复内容，需确认真实业务口径。
- 排放底单页面信息较少，需补充字段、模板与导出规范。
- 不同行业核算公式、因子来源与更新机制需明确责任部门。
- 交易数据是否对接外部交易所接口需确认（目前原型仅体现内部记录能力）。
- 单位与口径（吨CO₂、吨标煤、KWh、万元）需统一并形成数据字典。

## 11. 里程碑建议
1. 第一阶段（MVP）：登录、数据填报、核算配置、基础计算、报告列表。
2. 第二阶段：能碳总览、能耗/用电分析、预警能力。
3. 第三阶段：碳资产总览、配额管理、交易管理与深度分析。
4. 第四阶段：外部系统对接、规则引擎优化、管理驾驶舱增强。
